# 3.7 分析表与冲突处理

#### 目标

`BuildParsingTable` 从 LALR 状态和前瞻符生成 `ACTION`/`GOTO` 表，同时定义 `LALRConflictHandler` 接口和 `DefaultConflictHandler` 默认实现（比较优先级→结合性决策）。`LanguageSetter` 抽象输出接口供外部消费分析结果。

#### 实现

- `ConflictAction`/`ActionType` 枚举，`Action` 结构体 `{type, target}`
- `LALRConflictHandler`：`HandleShiftReduce`（含两方优先级/结合性参数）、`HandleReduceReduce`
- `DefaultConflictHandler`：高优先级胜出，同优先级按结合性——`kLeft`→规约、`kRight`→移入、`kNonAssoc`→`kAbort`
- `BuildParsingTable`：从 `lalr_states_.transitions` 填 SHIFT/GOTO，从 `lr0_lookaheads_` 按 `lr0_to_lalr_` 填 REDUCE；`[ROOT -> S·]` 填 ACCEPT；冲突时调用 handler
- `LanguageSetter`：输出产生式、符号表、LALR状态、ACTION/GOTO表
