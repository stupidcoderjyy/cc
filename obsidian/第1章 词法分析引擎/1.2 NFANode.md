# 1.2 NFANode

#### 目标

在已搭建的项目骨架基础上，在`src/lex/`目录下创建NFA节点类，实现：自动递增的节点ID用于哈希和相等比较、三种边类型（无转移、单epsilon转移、双epsilon转移、字符转移）的枚举、epsilon边添加的自动类型升级逻辑、字符边绑定`CharPredicate`断言函数、接受状态标记，以及用于调试的`ToString`字符串化方法。

#### 实现

1. 定义NFANode类和边类型枚举
    1. 在`src/lex/nfa_node.h`中定义`namespace cc`下的`NFANode`类，使用`#ifndef CC_NFA_NODE_H_`头文件保护
    2. 在类内定义`enum class EdgeType : uint8_t`枚举，包含`kNoEdge = 0`、`kSingleEpsilon = 1`、`kDoubleEpsilon = 2`、`kChar = 3`四个值
    3. 声明一个`static int node_count_`静态成员用于全局节点计数器，每个节点构造时自增并赋值给`const int id_`成员
    4. 定义`using CharPredicate = std::function<bool(int)>`类型别名，作为字符转移边的断言函数类型

2. 实现epsilon边添加逻辑
    1. 实现`AddEpsilonEdge(NFANode* next)`：若当前`edge_type_`为`kNoEdge`则将`next`赋给`next1_`并切换为`kSingleEpsilon`；若当前为`kSingleEpsilon`则将`next`赋给`next2_`并切换为`kDoubleEpsilon`，实现从无到单再到双的自动类型升级
    2. 实现重载`AddEpsilonEdge(NFANode* next1, NFANode* next2)`：直接同时设置`next1_`和`next2_`并强制切换为`kDoubleEpsilon`类型

3. 实现字符转移边
    1. 实现`AddCharEdge(CharPredicate predicate, NFANode* next)`：将断言函数通过`std::move`存入`predicate_`成员，将目标节点存入`next1_`，边类型设为`kChar`

4. 实现调试和比较功能
    1. 实现`ToString()`方法：根据`edge_type_`分别输出`id(null)`、`id(en1)`、`id(en1,en2)`、`id(cn1)`格式的字符串
    2. 实现`operator==`和`operator!=`：仅基于`id_`比较，保证每个节点具有唯一标识
    3. 在`src/lex/nfa_node.cpp`中初始化静态成员`int NFANode::node_count_ = 0`
