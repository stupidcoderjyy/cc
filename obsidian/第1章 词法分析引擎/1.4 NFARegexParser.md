# 1.4 NFARegexParser

#### 目标

在NFA和NFANode模块之上实现正则表达式解析器，实现：递归下降解析器将正则表达式字符串编译为NFA图、支持`|`选择、隐式连接、`()`分组、`*`/`+`/`?`闭包、`[...]`字符类（含范围和排除）、`\`转义字符和`@`预定义字符类（`\d`/`\w`/`\a`/`\h`/`.`）、`Register`多次注册自动合并多条规则为单一NFA、以及配套gtest测试文件通过NFASimulator模拟NFA运行来验证所有解析结果的正确性。

#### 实现

1. 实现解析器主入口和规则注册
    1. 在`src/lex/nfa_regex_parser.h`中定义`class NFARegexParser`，持有`std::unique_ptr<AbstractInput> input_`作为输入抽象和`std::vector<NFA> nfa_pool_`池维持所有中间NFA的生命周期
    2. 实现`Register(const std::string& regex, const std::string& token)`：创建`StringInput`包装正则字符串，调用`Expr(false)`递归解析，在返回的NFA的`end`节点上通过`set_accepted(true)`标记接受状态并在`node_id_to_token_`中记录token名称
    3. 处理多条规则合并：首次注册时直接将`nfa_start_`/`nfa_end_`指向第一个NFA的入口出口；后续注册时创建新包装节点`newStart`，从该节点添加双epsilon边分别指向新规则的start和既有NFA的start，实现"或"语义的规则并联

2. 实现递归下降解析器（Expr / Seq / Atom三层结构）
    1. 实现`Expr(bool group)`：循环读取字符，遇到`|`时调用`Seq()`解析右侧子表达式并执行`result.Or()`合并；遇到`)`时若不在group内则报错"unclosed group"，否则返回当前结果；其他字符走`default`分支调用`result.And(Seq())`执行连接。group为true时若读完所有输入仍未遇到`)`则报错
    2. 实现`Seq()`：先`Retract()`回退一个字符，循环读取——遇到`(`则递归`Expr(true)`并用`CheckClosure`处理后续闭包符；遇到`|`或`)`则回退并返回；其他字符走`Atom()`处理单字符匹配
    3. 实现`Atom()`：先回退再读取，根据首字符分派：`[`进入`Clazz()`字符类解析，`\`进入单字符转义，`@`进入`Escape()`预定义类，`*`/`?`/`+`报错"invalid closure symbol"，普通字符直接生成单字符断言。解析完成后创建临时NFA并调用`CheckClosure`处理可能紧跟在后的闭包运算符

3. 实现闭包检测和字符类解析
    1. 实现`CheckClosure(NFA target)`：向前看一个字符，若为`*`则调用`target.Star()`、`+`则`target.Plus()`、`?`则`target.Quest()`，否则回退，返回处理后的NFA
    2. 实现`Clazz()`：循环读取`[...]`内部字符，遇到嵌套`[`递归调用自身解析内层字符类并用lambda组合器`[](int c) { return a(c) || b(c); }`做OR合并；遇到`]`返回累积的断言；其他字符调用`MinClazzPredicate()`解析单个字符类元素
    3. 实现`MinClazzPredicate()`：区分三种情况——以`^`开头时收集排除集合返回`[excluded](int c) { return !excluded->count(c); }`排除断言；`a-z`格式返回范围断言`[b, b2](int c) { return c >= b && c <= b2; }`；普通单字符返回相等断言

4. 实现转义序列
    1. 实现`Escape()`：根据`@`后字符分派——`d`/`D`返回数字断言`[0-9]`，`w`/`W`返回单词字符断言`[a-zA-Z0-9_]`，`u`/`U`返回字母数字断言`[a-zA-Z0-9]`，`h`/`H`返回十六进制断言`[0-9a-fA-F]`，`a`/`A`返回字母断言`[a-zA-Z]`，`.`返回全匹配断言`[](int) { return true; }`

5. 编写gtest测试文件
    1. 在`test/test_nfa.cpp`中实现`NFASimulator`辅助类：通过epsilon闭包BFS和字符转移模拟NFA运行，提供`Match(input)`方法返回布尔匹配结果
    2. 编写`NFARegexParserTest`测试夹具类，提供静态`MatchRegex`辅助方法简化单条正则测试
    3. 编写基础字符匹配测试：`SingleChar`验证单字符精确匹配和空串拒绝，`Concatenation`验证多字符串联
    4. 编写运算符测试：`Or`验证`|`选择（含多字符分支），`Star`/`Plus`/`Quest`验证闭包语义边界（零次、一次、多次）
    5. 编写字符类测试：`CharClass`验证`[...]`和范围`[a-z]`及排除`[^...]`，`CharClassNested`验证嵌套字符类
    6. 编写转义测试：`Escape`验证`\d`/`\w`等预定义类
    7. 编写分组和复合测试：`Group`验证`(...)`分组与闭包叠加，`Complex`验证多运算符混合
    8. 编写多规则并联测试：`MultipleRegister`验证两次Register后的NFA能同时匹配两条规则
    9. 编写异常输入测试：`InvalidRegex`验证未闭合括号、多余括号、空选择分支、闭包符号错位等非法正则抛出`std::exception`
    10. 将`test_nfa.cpp`加入`test/CMakeLists.txt`的源文件列表，确保`gtest_discover_tests`自动发现并运行所有测试用例
