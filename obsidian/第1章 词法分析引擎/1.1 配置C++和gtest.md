# 1.1 配置C++和gtest

#### 目标

在空项目中搭建完整的C++构建与测试基础设施，实现：顶层`CMakeLists.txt`配置C++23标准和项目结构，引入googletest作为测试框架，建立`src/`源代码目录和`test/`测试目录的CMake子项目，确保一个简单的`TEST(HelloTest, BasicAssertions)`测试用例能够通过Ninja编译并执行。

#### 实现

1. 创建项目顶层CMake构建文件
    1. 在项目根目录下新建`CMakeLists.txt`，设置`cmake_minimum_required(VERSION 4.0)`和`project(cc LANGUAGES CXX)`
    2. 将C++标准设置为C++23，启用`CMAKE_CXX_STANDARD_REQUIRED ON`以保证编译器必须支持
    3. 配置Debug和Release的编译选项：Debug使用`-O0 -g`，Release使用`-O2 -DNDEBUG`
    4. 将MSYS2 MinGW的库路径`C:/msys64/mingw64`添加到`CMAKE_PREFIX_PATH`前缀，以便CMake能找到googletest等第三方库
    5. 通过`add_subdirectory`将`thirdparty`、`src`、`test`三个子目录纳入构建

2. 引入googletest测试框架
    1. 在`thirdparty/`目录下通过git submodule添加googletest的`v1.17.x`分支
    2. 在`thirdparty/CMakeLists.txt`中用`add_subdirectory(googletest)`将gtest纳入项目构建
    3. 设置`BUILD_GMOCK OFF`和`INSTALL_GTEST OFF`以关闭非必要的gtest组件，加快构建速度

3. 建立src源代码目录结构
    1. 在`src/CMakeLists.txt`中创建静态库目标`cc_core`并设置别名`cc::core`
    2. 在`src/test.h`中声明一个`int TestNum()`函数作为基础连通性验证
    3. 在`src/test.cpp`中实现`TestNum()`返回常量10
    4. 将`test.cpp`加入`cc_core`库的源文件列表

4. 建立test测试目录结构
    1. 在`test/CMakeLists.txt`中调用`enable_testing()`和`include(GoogleTest)`激活CTest与gtest集成
    2. 创建可执行文件目标`cc_test`并链接`cc::core`和`GTest::gtest_main`
    3. 在`test/test_gtest.cpp`中编写`TEST(HelloTest, BasicAssertions)`测试用例，验证`TestNum()`返回10且基本断言`EXPECT_TRUE(true)`通过
    4. 通过`gtest_discover_tests`自动注册所有gtest测试用例到CTest
