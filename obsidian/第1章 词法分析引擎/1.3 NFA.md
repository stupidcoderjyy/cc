# 1.3 NFA

#### 目标

在NFANode基础上构建NFA子图组合类，实现：使用`unique_ptr`持有NFA子图的入口和出口节点以明确所有权语义、`And`（连接）和`Or`（选择）运算消费另一个NFA的所有权、`Star`（星闭包）/`Plus`（正闭包）/`Quest`（可选）三种一元闭包运算、`AndAtom`快捷方法添加单个字符断言节点、以及内部节点池（`internal_nodes_`）维持那些不再是根节点但仍在图内被引用的节点的生命周期。

#### 实现

1. 定义NFA类和所有权策略
    1. 在`src/lex/nfa.h`中定义`class NFA`，成员`start_`和`end_`使用`std::unique_ptr<NFANode>`存储，独占子图所有权
    2. 添加`std::vector<std::unique_ptr<NFANode>> internal_nodes_`成员作为内部节点池，存放应用闭包运算或`And`/`Or`后不再是入口/出口的中间节点
    3. 删除拷贝构造和拷贝赋值，声明默认移动构造和移动赋值，确保NFA实例只能移动不能复制
    4. 提供`IsEmpty()`查询方法：`start_`或`end_`中任一为空则返回true

2. 实现二元组合运算
    1. 实现`And(NFA other)`：若当前为空则直接接管`other`的`start_`、`end_`和`internal_nodes_`；否则从当前`end_`添加epsilon边到`other.start_`，将旧`end_`和`other.start_`移入`internal_nodes_`池维持生命，最终将`end_`替换为`other.end_`
    2. 实现`Or(NFA other)`：若当前为空则同`And`空处理；否则创建新的`newStart`和`newEnd`包装节点，从`newStart`分别向当前`start_`和`other.start_`添加双epsilon边，从当前`end_`和`other.end_`分别向`newEnd`添加epsilon边，所有旧根节点（4个）移入`internal_nodes_`池
    3. 实现私有方法`MergeInternalNodes(NFA& other)`：使用`std::make_move_iterator`将`other`的内部节点池合并到当前池

3. 实现一元闭包运算
    1. 实现`Star()`：创建`newStart`和`newEnd`包装节点，从`newStart`添加双epsilon边指向原`start_`和`newEnd`实现零次匹配跳过，从原`end_`添加双epsilon边指回`start_`和`newEnd`实现循环，旧`start_`和`end_`移入内部池
    2. 实现`Plus()`：创建`newStart`和`newEnd`，从`newStart`添加epsilon边到原`start_`保证至少匹配一次，从原`end_`添加双epsilon边指回`start_`和`newEnd`实现循环，旧根节点移入内部池
    3. 实现`Quest()`：创建`newStart`和`newEnd`，从`newStart`添加双epsilon边指向原`start_`和`newEnd`实现零次或一次选择，从原`end_`添加epsilon边到`newEnd`，旧根节点移入内部池

4. 实现工具方法
    1. 实现`AndAtom(CharPredicate predicate)`：创建`newStart`和`newEnd`节点，在`newStart`上调用`AddCharEdge`绑定断言，之后按`And`逻辑连接到当前NFA
    2. 实现`ToString()`：空时返回`"null"`，否则返回`start_->ToString() + ", " + end_->ToString()`格式
    3. 提供`start()`和`end()`访问器返回裸指针`NFANode*`，供外部遍历NFA图使用
