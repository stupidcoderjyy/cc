# 2.6 DFASetter与集成测试

#### 目标

定义DFASetter抽象输出接口，在DFABuilder中实现Build一体化流程（BuildCharClassMap → ComputeNfaCharClassMask → BuildDfaFromNfa → MinimizeDfa → OutputData），实现CollectingDfaSetter测试桩类收集输出数据，编写端到端gtest覆盖OrAB、StarA、ConcatAB、StarAB、StarAOrStarB五种正则的完整NFA→DFA→最小化→输出流程。

#### 实现

1. 定义DFASetter抽象接口
    1. 在src/lex/dfa_setter.h中定义纯虚类DFASetter，析构函数为virtual default
    2. 声明6个纯虚方法：SetCharClassCount(count)、SetCharToClass(char_to_class)、SetDfaStatesCount(count)、SetStartState(id)、SetStateInfo(stateId, isAccepted, token)、SetTransition(start, input, target)
    3. 声明虚方法Finish()（默认空实现），供派生类在全部设置完成后做收尾操作

2. 实现DFABuilder::OutputData输出方法
    1. 调用setter.SetCharClassCount(class_count_)和SetCharToClass(char_to_class_)
    2. 调用SetDfaStatesCount和SetStartState
    3. 遍历DFA所有状态：对每个状态依次调用SetStateInfo设置接受标志和token
    4. 对每个状态遍历所有字符类：仅当class_id_to_next[cid] != -1时调用SetTransition输出有效转移
    5. 最后调用setter.Finish()触发收尾

3. 实现DFABuilder::Build一体化流程
    1. 依次调用BuildCharClassMap() → ComputeNfaCharClassMask() → BuildDfaFromNfa() → MinimizeDfa(raw) → OutputData(minimized, setter)
    2. 对外暴露单一入口，简化调用方使用

4. 编写CollectingDfaSetter测试桩与集成测试
    1. 在test/lex/dfa_builder_test.cpp中实现CollectingDfaSetter：派生DFASetter，各虚方法将参数存入public成员变量/映射表
    2. 提供ExpectStateCount/ExpectStartState/ExpectStateAccepted/ExpectTransition/ExpectNoTransition断言辅助方法
    3. OrAB测试：验证char_class_count=3，state_count=2（最小化后），起始状态对a和b转到同一接受态
    4. StarA测试：验证压缩为1个状态且为接受态，对a自循环
    5. ConcatAB测试：验证3个状态的链式转移结构
    6. StarAB测试：验证(a|b)*压缩为1个状态，对a和b均自循环
    7. StarAOrStarB测试：验证a*|b*压缩后start对a和b分别转到不同接受态，各接受态仅对其字符自循环、token一致
