# 2.4 子集构造法

#### 目标

在DFABuilder中实现Next驱动函数和BuildDfaFromNfa的worklist子集构造算法，实现：给定字符类和当前NFA状态组，Next通过Char边转移并计算epsilon闭包得到下一状态组；BuildDfaFromNfa从根节点出发用worklist队列逐步发现所有可达的DFA状态并为每个状态填充class_id_to_next转移表。配套gtest覆盖单字符、含epsilon、选择分支和连接四种NFA到DFA的构造结果。

#### 实现

1. 实现Next(char_class, group)转移驱动
    1. 创建空NFAGroup result，调用VisitNfaGroup遍历group内每个NFA节点
    2. 只处理edge_type_ == kChar且其nfa_node_to_char_class_[node->id()].test(char_class)为真的节点
    3. 将匹配节点的next1()->id()置位到result，最后返回EpsilonClosure(result)——转移后立即计算epsilon闭包

2. 实现BuildDfaFromNfa()子集构造法
    1. 创建初始DFA：为root_node_构造单元素NFAGroup并调用EpsilonClosure得到start_nfa_group，用CreateDfaState创建起始DFA状态并入栈unprocessed
    2. 在BitsetMap<DFAState::id_t, kMaxNfaGroups>中维护已发现的nfa_group→dfa_state_id映射
    3. worklist主循环：弹出栈顶DFA状态，对该状态的nfa_group针对每个字符类cid调用Next(cid, group)
    4. 若next_nfa_group非空（有实际转移）且未曾映射则调用CreateDfaState新建DFA状态并入栈，将cid→next_id填入当前状态的class_id_to_next[cid]

3. 实现CreateDfaState辅助方法
    1. new_id取dfa.states.size()作为DFA状态ID
    2. 调用VisitNfaGroup遍历nfa_group内的NFA节点，若任一节点的accepted()为真则将DFA状态标记为接受态并取对应token，存入node_id_to_token_[node->id()]
    3. 创建DFAState对象并用new_id、nfa_group、accepted标志和token构造，class_id_to_next初始化为全-1（失效）

4. 编写gtest测试
    1. 在test/lex/dfa_builder_build_dfa_test.cpp中实现MakeBuilderFromParser辅助：解析正则后调用BuildCharClassMap+ComputeNfaCharClassMask
    2. SingleCharA测试："a"产生2个DFA状态，start对a类转到状态1且状态1为接受态
    3. OrAB测试："a|b"产生3个状态，start对a和b分别转到两个不同的接受态
    4. StarA测试："a*"至少2个状态，start为接受态，对a的自循环指向自身或另一接受态
    5. ConcatAB测试："ab"产生3个状态链——start(a)→mid(b)→accept
