# 2.4 子集构造法

#### 目标

实现 `Next(char_class, NFAGroup)` 和 `BuildDfaFromNfa()`，通过worklist队列从根节点的epsilon闭包出发逐步发现所有可到达的DFA状态。

#### 实现

1. `Next(char_class, group)` 转移驱动
    1. 遍历group内节点，筛选 `edge_type == kChar` 且 `nfa_node_to_char_class_[id].test(char_class)` 为真
    2. 将匹配节点的 `next1()->id()` 置位到新NFAGroup，调用 `EpsilonClosure` 后返回

2. `BuildDfaFromNfa()` worklist
    1. 初始状态：`root_node_` 单元素NFAGroup的epsilon闭包 → `CreateDfaState` 创建state 0
    2. `BitsetMap<id_t, kMaxNfaGroups>` 维护 `NFAGroup → DFAState.id` 映射
    3. worklist循环：弹出DFA状态，对该状态的nfa_group针对每个字符类 `cid` 调用 `Next`
    4. 若新NFAGroup非空且未见过，`CreateDfaState` 新建 → 入栈；`class_id_to_next[cid] = target`

3. `CreateDfaState`：遍历nfa_group，若任NFA节点 `accepted()` 则DFA状态标记接受态
