# 3.4 LR(0)项集族构建

#### 目标

构建完整的 `LR(0)` 状态机（`Canonical Collection`），包含所有状态和转移边。从 `ROOT -> ·S`（产生式0）的闭包出发，通过 `BFS` 队列对每个状态的所有圆点后符号求 `GotoFunc`，记录新发现的项集和状态间转移边。语法不再需要手动增广——`Syntax` 首次调用 `AddProduction` 时已自动插入 `ROOT -> S` 作为产生式0。

#### 实现

1. 定义 `LR0State` 结构体
    1. 在 `src/syntax/lalr_parser.h` 中定义 `struct LR0State { std::set<Item> items; std::map<int, int> transitions; }`
    2. `items` 存储该状态的 `LR(0)` 项集，`transitions` 以 `symbolId -> targetStateIndex` 记录 `GOTO` 转移

2. 实现 `BuildCanonicalCollection()`
    1. 使用 `std::map<std::set<Item>, int>` 作为已见项集到状态索引的映射（依赖 `std::set<Item>` 自带的 `operator<`）
    2. 创建初始状态 `start_items = Closure({{0, 0}})`（`ROOT -> ·S`），存入 `states_[0]`，索引0入队 `worklist`
    3. `BFS` 主循环：出队当前状态索引 `cur`，遍历 `states_[cur].items` 收集所有圆点后的符号编号（用 `std::set<int>` 去重）
    4. 对每个符号 `sym_id` 调用 `GotoFunc` 得到新项集——若非空则在 `seen` 中查找是否已存在；若不存在则创建新状态、分配索引、入队；在 `states_[cur].transitions[sym_id]` 记录转移目标索引
    5. 构造函数中自动调用 `BuildCanonicalCollection()`，确保 `LALRBuilder` 构造完毕时项集族已就绪

3. 编写 `gtest` 验收
    1. `StateCount`：状态数在 5~8 范围内
    2. `StartStateContainsRootItems`：状态0包含 `ROOT`/`E`/`T` 全部初始项
    3. `StartStateTransitions`：状态0对 `E`/`T`/`id` 均有转移且目标不同
    4. `StateAfterEHasPlusTransition`：沿 `E` 到达的状态对 `+` 有转移
    5. `ReduceStatesHaveNoTransitions`：含 `E -> T·` 的状态无转移边
    6. `NoDuplicateStates`：所有状态的项集互不相同
