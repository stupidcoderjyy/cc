# 3.3 LR(0)闭包与GOTO

#### 目标

实现 `LR` 分析器的两个基础运算——闭包（`Closure`）和 `GOTO` 函数，不涉及前瞻符。`closure` 将圆点后为非终结符的项扩展为该非终结符的所有产生式；`GOTO` 对一个项集沿指定符号移动圆点后求闭包。

#### 实现

1. 实现 `closure(const std::set<Item>& items)`
    1. 创建结果集 `closed = items`，使用工作队列（`std::queue` 或栈）依次处理
    2. 弹出当前项 `[A -> α·Bβ]`，若圆点后符号 `B` 为非终结符，则遍历该非终结符的所有产生式 `B -> γ`，将项 `[B -> ·γ]`（圆点在最左，`dotPos=0`）加入 `closed`（若尚未出现）
    3. 循环直到工作队列为空，返回 `closed`

2. 实现 `gotoFunc(const std::set<Item>& items, const Symbol& sym)`
    1. 创建目标集 `moved`，遍历 `items` 中每个项 `[A -> α·Xβ]`——若圆点后符号 `X == sym`，则将圆点右移一位的项 `[A -> αX·β]` 加入 `moved`
    2. 对 `moved` 调用 `closure()`，返回闭包后的结果
    3. 验收示例：给定初始项集 `{E'' -> ·E}`，对符号 `E` 调用 `GOTO`，应得到 `{E'' -> E·, E -> E·+T}`（假设文法存在 `E -> E + T` 产生式）

3. 注意
    1. `Item` 需要支持 `operator<` 和 `operator==` 以放入 `std::set`，比较逻辑为 `prodId` 不等则比 `prodId`，否则比 `dotPos`
    2. 此阶段仅验证 `LR(0)` 核心运算正确性，不涉及前瞻符
