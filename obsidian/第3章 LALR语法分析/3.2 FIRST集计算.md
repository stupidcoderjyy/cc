# 3.2 FIRST集计算

#### 目标

在 `LALRParser` 中实现语法分析器最基础的数学工具——`FIRST` 集，采用迭代不动点算法，处理空产生式。`FIRST(X)` 定义为从符号 `X` 出发能推导出的所有符号串的首个终结符的集合（若 `X` 可推导出空串则包含 `epsilon`）。

#### 实现

1. 定义 `LRItem` 内部类型和 `FIRST` 集数据结构
    1. 在 `src/syntax/lalr_parser.h` 的 `LALRBuilder` 类内定义 `struct Item { int prodId; int dotPos; }`，表示"产生式ID + 圆点位置"的 `LR` 项
    2. 添加私有成员：`std::vector<std::set<Symbol>> first_sets_`（按符号编号索引每个符号的 `FIRST` 集）、`UnorderedSymbolMap<int> symbol_to_id_`（符号到内部编号的映射，便于数组索引）、`std::vector<Symbol> id_to_symbol_`（反向映射）

2. 实现 `computeFirstSets()` 不动点算法
    1. 初始化：遍历所有终结符和 `EOF` 符号，将其 `FIRST` 集设为自身单元素集合（即 `FIRST(a) = {a}`），非终结符的 `FIRST` 集初始为空
    2. 迭代主循环：遍历所有产生式 `A -> X1 X2 ... Xn`，对每个右部符号 `Xi` 依次处理——将 `FIRST(Xi)` 中除 `epsilon` 外的所有符号加入 `FIRST(A)`；若 `FIRST(Xi)` 包含 `epsilon` 则继续处理 `Xi+1`，否则停止；若所有右部符号的 `FIRST` 都包含 `epsilon`，则 `FIRST(A)` 也包含 `epsilon`
    3. 重复迭代直到所有非终结符的 `FIRST` 集不再变化（不动点）

3. 编写 `gtest` 验收
    1. 构建文法 `A -> B c, B -> d | epsilon`（空产生式即 `rhs` 为空的 `Production`）
    2. 验证 `FIRST(B) = {d, epsilon}`、`FIRST(A) = {d, c}`（因为 `B` 可推出 `epsilon`，所以 `A` 的首个终结符可以是 `B` 推出的 `d` 或是跳过 `B` 后直接取的 `c`）
