# 3.7 分析表与冲突处理

#### 目标

从 LALR 合并后的状态和已计算的前瞻符生成 `ACTION`/`GOTO` 分析表，并通过 `LALRErrorHandler` 接口处理移入-规约和规约-规约冲突。`ACTION` 表为 `LALR状态 × 终结符 → 动作`（`SHIFT`/`REDUCE`/`ACCEPT`/`ERROR`），`GOTO` 表为 `LALR状态 → map<非终结符ID, 目标状态>`。

#### 实现

1. 定义 `LALRErrorHandler` 抽象接口和 `DefaultErrorHandler`
    1. 在 `src/syntax/lalr_error_handler.h` 中定义 `enum class ConflictAction { kShift, kReduce, kAbort }`、`enum class ActionType`、`Action` 结构体
    2. `LALRErrorHandler` 纯虚接口：`HandleShiftReduce` 接收各项优先级/结合性参数，`HandleReduceReduce` 接收候选产生式列表
    3. `DefaultErrorHandler`：比较规约产生式与 lookahead 的优先级，高者胜出；同优先级按结合性决策（`kLeft`→规约，`kRight`→移入，`kNonAssoc`→放弃）

2. 实现 `BuildParsingTable(LALRErrorHandler* handler)`
    1. 从 `lalr_states_` 的 `transitions` 填充 `SHIFT`（目标为终结符）和 `GOTO`（目标为非终结符）
    2. 从 `lr0_lookaheads_` 按 `lr0_to_lalr_` 合并到对应 LALR 状态，对每个归约项（圆点在末尾）的每个前瞻符填充 `REDUCE`
    3. 对 `[ROOT -> S·]` 且前瞻符含 `$` 填充 `ACCEPT`
    4. 冲突检测：`SHIFT-REDUCE` 调 `HandleShiftReduce`，`REDUCE-REDUCE` 调 `HandleReduceReduce`；`handler` 为 `nullptr` 时 `SHIFT-REDUCE` 默认保留 `SHIFT`，`REDUCE-REDUCE` 抛异常

3. 编写 `gtest` 验收
    1. `NoConflictsWithPriority`：完整表达式文法设置 `+`/`-` 和 `*`/`/` 的优先级后无冲突生成
    2. `TableHasEntries`：分析表非空
