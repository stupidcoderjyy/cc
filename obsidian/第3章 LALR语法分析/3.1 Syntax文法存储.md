# 3.1 Syntax文法存储

#### 目标

实现 `Syntax` 类的全部数据存储和优先级配置接口（不含算法）。首次调用 `AddProduction` 时自动插入 `ROOT -> head` 作为产生式 0，后续调用不重复。符号属性支持三种粒度设置，产生式优先级可独立覆盖。

#### 实现

1. 实现 `AddProduction` 产生式添加
    1. 核心重载 `AddProduction(head, vector<Symbol>)`：调用 `RegisterSymbol` 将 `head` 和 `body` 中各符号归入 `terminals_`/`non_terminals_`
    2. 首次调用时调用 `InsertRootProduction(head)` 插入 `ROOT -> head`（`id=0`，`priority=0`，`assoc=kLeft`）
    3. 用 `next_prod_id_` 自增生成唯一 `id`，`emplace_back` 就地构造 `Production`

2. 实现符号属性设置与查询
    1. `SetSymbolPriority`/`SetSymbolAssociativity`/`SetSymbolProperties`：根据 `sym.type` 选择 `terminals_` 或 `non_terminals_`，`find` 命中则修改已有符号，否则 `emplace` 新符号
    2. `FindSymbol(name, type)`：构造临时 `key` 在对应 map 查找，返回 `const Symbol*` 或 `nullptr`
    3. `root_symbol()` 固定返回 `{"ROOT", kNonTerminal}`，`end_symbol()` 固定返回 `{"$", kEof}`

3. 实现产生式属性覆盖
    1. `SetProductionPriority`/`SetProductionAssociativity` 通过 `ranges::find_if` 按 `prodId` 查找并修改

4. 当前 gtest 覆盖率（`syntax_test.cpp`）
    1. `RootProductionAutoInserted`：ROOT 符号恒存在，首次 AddProduction 后自动插入
    2. `RootProductionOnlyOnce`：后续 AddProduction 不重复插入
    3. `SymbolPriorityAndAssociativity`：符号属性设置和查询
    4. `ProductionPriorityOverride`/`ProductionPriorityOverrideOnlyTarget`：产生式属性覆盖及隔离
    5. `FindSymbolNotFound`/`SetSymbolPropertiesUpdate`：查找与更新
    6. `EmptyProduction`/`NonTerminalPriority`/`EofSymbolFiltered`/`AddProductionCollectsSymbols`
