# 3.1 Syntax文法存储

#### 目标

实现 `Syntax` 类的全部数据存储和优先级配置接口（不含算法），实现：`AddProduction` 双重重载自动维护 `terminals_`/`non_terminals_` 符号集合、`SetStartSymbol` 设置文法的起始符号、`SetSymbolPriority`/`SetSymbolAssociativity`/`SetSymbolProperties` 三个粒度配置符号属性、`SetProductionPriority`/`SetProductionAssociativity` 覆盖特定产生式优先级、`FindSymbol` 按名称和类型检索符号。验收标准：添加表达式文法 `E -> E + T`，设置 `+` 优先级为 `1` 左结合，查询结果正确（已通过 `syntax_test.cpp` 的6个 `gtest` 用例）。

#### 实现

1. 实现 `AddProduction` 产生式添加
    1. 提供 `AddProduction(lhs, std::initializer_list<Symbol>)` 代理重载转为 `vector` 后调用核心实现
    2. 核心实现调用 `AddSymbol(lhs)` 将左部归入非终结符集合，遍历 `rhs` 对每个符号调用 `AddSymbol` 按 `SymbolType` 自动归入 `terminals_` 或 `non_terminals_`
    3. 用 `next_prod_id_` 自增生成唯一 `ID`，从 `lhs` 继承 `priority` 和 `assoc` 作为产生式默认值，推入 `productions_` vector 并返回 `ID`

2. 实现符号属性设置与查询
    1. `SetSymbolPriority`/`SetSymbolAssociativity`/`SetSymbolProperties`：根据 `sym.type` 选择 `terminals_` 或 `non_terminals_` 的引用，若 `find` 命中则直接修改已存在符号的属性字段，否则以 `sym` 为蓝本构造新 `Symbol` 设置属性后 `emplace`
    2. `FindSymbol`：构造临时 `key`（仅 `name`+`type`），在对应 `map` 中查找返回 `const Symbol*` 指针，未找到返回 `nullptr`
    3. `SetStartSymbol` 将传入符号赋给 `start_symbol_`，同时调用 `AddSymbol` 确保纳入非终结符集合

3. 实现产生式属性覆盖和辅助方法
    1. `SetProductionPriority`/`SetProductionAssociativity` 通过 `std::ranges::find_if` 按 `prodId` 查找，修改命中产生式的对应字段
    2. `AddSymbol` 私有方法：跳过 `kEof` 类型符号，根据 `type` 选择 `map` 执行 `emplace`（已存在则保留原属性）
    3. 提供 `productions()`、`terminals()`、`non_terminals()`、`start_symbol()`、`end_symbol()` 系列 `getter`——`end_symbol_` 固定为 `{"$", SymbolType::kEof}`

4. 当前 `gtest` 覆盖率
    1. `AddProductionAndRetrieve`：6条产生式构建表达式文法，验证 `size` 和首条 `id`/`lhs`/`rhs`
    2. `StartSymbol`：`SetStartSymbol` 后 `start_symbol()` 验证
    3. `SymbolPriorityAndAssociativity`：`SetSymbolProperties` 设置 `+`/`*` 优先级后 `FindSymbol` 验证
    4. `ProductionPriorityOverride`：`SetProductionPriority`/`SetProductionAssociativity` 覆写验证
    5. `FindSymbolNotFound`：不存在符号返回 `nullptr`
    6. `AddSymbolDuplicate`：重复添加同符号时属性更新为最新设置
