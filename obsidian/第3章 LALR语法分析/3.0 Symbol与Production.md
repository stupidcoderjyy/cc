# 3.0 Symbol与Production

#### 目标

定义文法的基础数据结构和工具类型，实现：`SymbolType` 三态枚举（`kTerminal`/`kNonTerminal`/`kEof`）、`Associativity` 三态枚举（`kLeft`/`kRight`/`kNonAssoc`）、`Symbol` 符号结构体（`name`/`type`/`priority`/`assoc`）及其 `operator==`、`operator<` 和 `SymbolHash` 哈希器、`UnorderedSymbolMap` 和 `OrderedSymbolMap` 模板别名、`Production` 产生式结构体（唯一 `ID`、左部、右部序列、优先级、结合性）。验收标准：能编译通过，能创建 `std::set<Symbol>` 和 `std::unordered_map<Symbol, int, SymbolHash>`。

#### 实现

1. 定义枚举类型
    1. 在 `src/syntax/production.h` 中定义 `enum class SymbolType { kTerminal, kNonTerminal, kEof }`，`kEof` 用于文法结束标记 `"$"`
    2. 定义 `enum class Associativity { kLeft, kRight, kNonAssoc }`，分别表示左结合、右结合、非结合（禁止同级运算符并存）

2. 定义 `Symbol` 符号结构体
    1. 定义 `struct Symbol` 包含成员：`std::string name`（符号名称）、`SymbolType type`（符号类型）、`int priority = 0`（数值越大优先级越高）、`Associativity assoc = Associativity::kLeft`（默认左结合）
    2. 提供构造函数：`Symbol(name, type, priority, assoc)` 四参数全指定、`Symbol(name, type)` 双参数默认值、默认构造函数
    3. 实现 `operator==`：按 `name` 和 `type` 比较（忽略 `priority` 和 `assoc`，同一语法符号可能在不同上下文中具有不同优先级属性但仍是同一个符号）
    4. 实现 `operator<`：先按 `type` 枚举比较再按 `name` 字典序，支持 `std::set<Symbol>` 的排序

3. 实现 `SymbolHash` 和映射模板
    1. 定义 `struct SymbolHash`，`operator()` 中组合 `std::hash<std::string>{}(s.name)` 和 `static_cast<int>(s.type) << 1` 的异或值
    2. 定义 `template <typename V> using UnorderedSymbolMap = std::unordered_map<Symbol, V, SymbolHash>` 类型别名——用于 `Syntax` 中 `terminals_` 和 `non_terminals_` 成员
    3. 定义 `template <typename V> using OrderedSymbolMap = std::map<Symbol, V, SymbolHash>` 类型别名——用于后续需要有序遍历符号集的场景（如符号编号、输出格式化）

4. 定义 `Production` 产生式结构体
    1. 定义 `struct Production` 包含成员：`int id`（唯一编号）、`Symbol lhs`（左部非终结符）、`std::vector<Symbol> rhs`（右部符号序列，空 `vector` 表示 `epsilon` 产生式）、`int priority`（产生式自身优先级，用于冲突解决覆盖符号默认值）、`Associativity assoc`（产生式结合性）
    2. 提供五参数构造函数 `Production(id, lhs, rhs, priority, assoc)`，优先级和结合性从 `lhs` 符号继承（在 `Syntax::AddProduction` 中传入）
