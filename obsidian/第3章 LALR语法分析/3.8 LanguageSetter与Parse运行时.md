# 3.8 LanguageSetter与Parse

#### 目标

定义 `LanguageSetter` 抽象输出接口作为 `Build` 流程的最后一步，将全部构造结果输出。`DebugParse` 调试专用方法使用分析表驱动栈式归约过程验证正确性。

#### 实现

1. `LanguageSetter` 抽象接口（`language_setter.h`）
    1. `SetProductions`/`SetTerminals`/`SetNonTerminals` — 输出产生式和符号表
    2. `SetLALRState(stateId, items)` — 输出每状态的项集
    3. `SetStateLookaheads(stateId, la)` — 输出每状态合并后的前瞻符表
    4. `SetAction(state, symbolId, type, target)` / `SetGoto(state, symbolId, target)` — 输出分析表
    5. `SetStartState(0)` / `Finish()` — 起始状态和收尾

2. `Build(LanguageSetter& setter)`
    1. 调用私有 `RunPipeline()` 执行全部 8 步构造（`InitSymbols → BuildParsingTable`）
    2. 最后调用私有 `OutputData(setter)` 输出全部数据

3. `OutputData` 输出逻辑
    1. 产生式从 `Syntax` 直接获取；符号表遍历 `terminals()`/`non_terminals()`
    2. LALR 状态遍历 `lalr_states_`，前瞻符按 `lr0_to_lalr_` 合并多个 LR(0) 状态的 lookahead
    3. ACTION 表遍历非 Error 单元格，GOTO 表遍历 map

4. `DebugParse(const vector<Symbol>& input)` 调试驱动
    1. 状态栈初始 `{0}`，循环：查 `action[state][lookahead]`
    2. `kShift` → 压入目标状态；`kReduce` → 弹出右部长度个状态，查 `goto_` 压入新状态；`kAccept` → 返回 `true`；`kError` → 返回 `false`

5. gtest 覆盖（`lalr_parser_parse_test.cpp`）
    1. `SingleToken`/`SimpleAddition`/`TwoAdditions` — 接受
    2. `SyntaxError`/`EmptyInputRejected`/`DoublePlus` — 拒绝
