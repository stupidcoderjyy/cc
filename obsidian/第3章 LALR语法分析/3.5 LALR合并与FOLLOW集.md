# 3.5 LALR合并与FOLLOW集

#### 目标

实现 `MergeLALRStates` 按项集核心合并 `LR(0)` 状态为 `LALR` 状态，并实现 `ComputeFollowSets` 不动点迭代计算所有非终结符的 `FOLLOW` 集。合并时按项的核心（`(prodId, dotPos)`，即忽略前瞻符后的部分）分组，同一核心组的状态合并为一个 LALR 状态，并重构 `GOTO` 转移表。`FOLLOW(A)` 为所有句型中可能紧跟在 `A` 后的终结符集合。

#### 实现

1. 实现 `MergeLALRStates()`
    1. 用 `std::map<std::set<Item>, int>` 以项集核心为键分组——核心相同的 `LR(0)` 状态映射到同一个 LALR 状态索引
    2. 遍历 `lr0_states_`：若核心已见过则记录 `lr0_to_lalr_[i] = 已有索引`，否则创建新 LALR 状态（项集 = 核心）并分配新索引
    3. 重构 `GOTO` 转移：遍历每条 `LR(0)` 转移边 `(sym_id, target)`，映射为 `lalr_states_[lalr_i].transitions[sym_id] = lr0_to_lalr_[target]`
    4. 若同状态下同一符号映射到不同目标，抛出 `"Grammar is not LALR"` 异常

2. 实现 `ComputeFollowSets()`
    1. 初始化：`FOLLOW(ROOT) = {$}`，其余为空
    2. 不动点迭代：遍历每条产生式 `A -> X1...Xn`，对每个非终结符 `Xi`，将 `FIRST(Xi+1...Xn)` 中所有终结符加入 `FOLLOW(Xi)`；若 `Xi+1...Xn` 全部可空则将 `FOLLOW(A)` 全部加入 `FOLLOW(Xi)`

3. 编写 `gtest` 验收
    1. `MergePreserveStateCount`：无同心状态的文法合并后状态数不变
    2. `LALRItemsMatchLR0`：LALR 状态项集与对应 LR(0) 一致
    3. `LALRTransitionsMatchLR0`：LALR 转移边映射正确
    4. `ExpressionGrammar`：`FOLLOW(E) = {+, $}`, `FOLLOW(T) = {+, $}`
    5. `BasicFollow`：`FOLLOW(B) = {c}`
    6. `MutualRecursionFollow`：互相递归时均含 `{$}`
    7. `AllNullable`：全可空链传播
    8. `TerminalHasEmptyFollow`：终结符 FOLLOW 为空
