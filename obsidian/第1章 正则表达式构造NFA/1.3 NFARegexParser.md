# 1.3 NFARegexParser

#### 目标

实现递归下降解析器将正则表达式字符串编译为NFA图。支持 `|` 选择、隐式连接、`()` 分组、`*`/`+`/`?` 闭包、`[...]` 字符类（含范围 `a-z`、取反 `[^...]`）、`\` 转义序列（`\d`/`\w`/`\a`/`\h`/`\n`/`\t`）。多条正则通过 `Register` 并联到同一个根节点。

#### 实现

1. 核心解析函数三层结构
    1. `Expr(bool group)`：循环读取，遇 `|` 则 `result.Or(Seq())`，遇 `)` 返回（group外报错）
    2. `Seq()`：遇 `(` 递归 `Expr(true)` 后 `CheckClosure`，遇 `|`/`)` 回退返回
    3. `Atom()`：根据首字符分派——`[`→`Clazz`，`\`→`Escape`，`*?+`→报错，其他→单字符断言；创建临时NFA后 `CheckClosure`

2. 字符类和转义
    1. `Clazz(bool exclude)`：循环读 `[...]` 内层，遇 `]` 退出，嵌套 `[` 递归，遇 `\` 走 `Escape`，其他走 `MinClazzPredicate`。`exclude` 为真时最终包装 `!inner(c)`
    2. `MinClazzPredicate()`：处理 `a-z` 范围、单字符等价
    3. `Escape()`：`d`→数字、`w`→字母数字下划线、`a`→字母、`h`→十六进制、`n`→`\n`、`t`→`\t`、`.`→全匹配

3. 规则注册与并联
    1. `Register(regex, token)`：创建 `StringInput` 包装正则，`Expr(false)` 解析，`SetAcceptedNode` 标记接受态
    2. 非首条规则创建新 `new_start` 双epsilon并联原 `root_node_` 和新NFA的 `start`

4. `RegisterSingles(chs)`：将所有单字符拼为 `[+*-...]` 字符类一次性注册，去重
