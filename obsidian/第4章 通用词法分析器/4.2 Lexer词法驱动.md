# 4.2 Lexer词法驱动

#### 目标

实现 `Lexer::NextToken(CompilerInput& ci)` 最长匹配算法：从起始状态出发沿字符类转移，追踪最后接受态，到达死态时回退多余字符，调用对应 `TokenSupplier` 生成 Token。

#### 实现

1. 初始化与空白跳过
    1. `ci.Skip(' ', '\r', '\n')` 跳过空白字符
    2. 检查 `ci.Available()`，无更多输入返回 `TokenEof`

2. DFA驱动循环
    1. `state` 从 `start_state_` 起，`last_accepted = -2`，`extraLoadedBytes = 0`
    2. 循环读取字节：若为负值尝试 `ReadUtf()`（UTF-8多字节字符统一视为特殊控制字符1）；查 `goto_[state][b]`；若转移为0（死态）则 `extraLoadedBytes` 递增并 break
    3. 若 `accepted_[state]` 为真，记录 `last_accepted = state` 并重置 `extraLoadedBytes`

3. Token生成
    1. 若无接受态或对应 `token_suppliers_` 为空 → `Approach('\r',' ','\t')` + 返回 `nullptr`
    2. 否则 `Retract(extraLoadedBytes)` 回退多余字符 → 调用 `token_suppliers_[last_accepted]()` → `OnMatched → kAccept` 即返回；`kPass` 则跳过继续；`kReject` 返回空指针
