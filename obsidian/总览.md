## 项目目标

本项目旨在从零构建一个可独立运行的编译器前端生成器。用户使用自定义DSL描述词法规则（正则表达式）和语法规则（上下文无关文法，含优先级/结合性），工具链自动生成对应的词法分析器（DFA）和LALR(1)语法分析器的C++代码。整个项目采用渐进式关卡设计，每一章聚焦一个编译器核心模块，确保每一步都有可编译、可测试的阶段性成果。

## 技术路线

词法分析阶段：正则表达式 → NFA构造（Thompson构造法） → 字符类签名合并压缩 → 子集构造法 → Hopcroft DFA最小化。语法分析阶段：Symbol/Production数据结构 → Syntax文法存储 → FIRST集不动点计算 → LR(0)闭包与GOTO → BFS构建项集族 → 同心项合并为LALR状态 → FOLLOW集计算 → 自发-传播前瞻符算法 → ACTION/GOTO分析表生成 → DefaultConflictHandler处理冲突。运行时层：Lexer（DFA驱动最长匹配+TokenSupplier工厂） → Parser（状态栈+符号栈+归约回调） → Generator（ScriptParserData解析DSL → Inja模板渲染C++代码）。整个工具链以C++23实现，构建系统CMake+Ninja，测试框架gtest。
